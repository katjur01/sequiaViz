change height of all cells in reactable shiny without header cells
2/2

To change the height of all table cells in a reactable in a Shiny app without affecting the header cells, you can use custom CSS. Here’s a step-by-step guide:

  1. Define Your CSS: Create a CSS class to target the table cells specifically. You can add this CSS to your Shiny app’s www directory or directly in the UI.

  2. Apply the CSS Class: Use the reactable library’s custom parameter to apply your CSS class to the table cells.

Here’s a basic example:

  1. Create a CSS file (e.g., styles.css in the www directory):

    css
    .custom-cell {
  height: 40px; /* Change this value to your desired cell height */
}

  2. Include the CSS in your Shiny app:

    r
    library(shiny)
library(reactable)

ui <- fluidPage(
  # Include custom CSS
  tags$head(
    tags$link(rel = "stylesheet", type = "text/css", href = "styles.css")
  ),
  reactableOutput("my_table")
)

server <- function(input, output) {
  output$my_table <- renderReactable({
    reactable(
      head(mtcars, 10),
      columns = list(
        everything = colDef(
          class = "custom-cell"
        )
      )
    )
  })
}

shinyApp(ui, server)

In this example, custom-cell is applied to all table cells. Adjust the CSS properties as needed to fit your design. The header cells will not be affected by this style.

Make sure to restart your Shiny app after making changes to the CSS for them to take effect.

create data.table where high_confidence is 1x hight, 4x medium, 44x low, 23x NA

To create a data.table in R with specific counts for each category (high_confidence, medium, low, and NA), you can use the data.table package. Here's how you can do it:

  1. Install and load the data.table package if you haven’t already:

    r
    install.packages("data.table")
library(data.table)

  2. Create the data.table with the desired counts:

    r
    library(data.table)

# Define counts
high_confidence_count <- 1
medium_count <- 4
low_count <- 44
na_count <- 23

# Create a vector with the values
values <- c(
  rep("high_confidence", high_confidence_count),
  rep("medium", medium_count),
  rep("low", low_count),
  rep(NA, na_count)
)

# Shuffle the values to randomize their order
set.seed(123)  # Set seed for reproducibility
values <- sample(values)

# Create the data.table
dt <- data.table(confidence = values)

# View the result
print(dt)

In this code:

  • rep() is used to create vectors with the specified counts of each category.
  • sample() shuffles the values to randomize their order.
  • data.table() creates the data.table with a single column named confidence.

vytvoř daata table kde jsou dva typy tkání: Blood vessel a no Brain. Pro blood vessel je 4x 5+, 2x6+ a pro no brain 5x5+, 1x6+

Chcete-li vytvořit data.table s dvěma typy tkání (Blood vessel a no Brain), kde každý typ tkáně má specifický počet hodnot, použijte následující postup:

  1. Instalujte a načtěte balíček data.table (pokud ještě není nainstalován):

    r
    install.packages("data.table")
library(data.table)

  2. Vytvořte data.table s požadovanými hodnotami:

    r
    library(data.table)

# Definujte počty pro každý typ tkáně
blood_vessel_5_plus <- 4
blood_vessel_6_plus <- 2
no_brain_5_plus <- 5
no_brain_6_plus <- 1

# Vytvořte vektor s hodnotami pro každý typ tkáně
blood_vessel_values <- c(
  rep("5+", blood_vessel_5_plus),
  rep("6+", blood_vessel_6_plus)
)
no_brain_values <- c(
  rep("5+", no_brain_5_plus),
  rep("6+", no_brain_6_plus)
)

# Přiřaďte typy tkání
tissue_types <- c(
  rep("Blood vessel", blood_vessel_5_plus + blood_vessel_6_plus),
  rep("no Brain", no_brain_5_plus + no_brain_6_plus)
)
values <- c(blood_vessel_values, no_brain_values)

# Zamíchejte hodnoty pro náhodné pořadí
set.seed(123)  # Nastavte semeno pro reprodukovatelnost
shuffled_indices <- sample(length(values))
values <- values[shuffled_indices]
tissue_types <- tissue_types[shuffled_indices]

# Vytvořte data.table
dt <- data.table(
  tissue_type = tissue_types,
  value = values
)

# Zobrazte výsledek
print(dt)

V tomto kódu:

  • rep() se používá k vytvoření vektorů s požadovanými počty pro každý typ tkáně.
  • c() sloučí hodnoty do jednoho vektoru.
  • sample() zamíchá hodnoty a přiřadí je k odpovídajícím typům tkání.
  • data.table() vytvoří data.table se dvěma sloupci: tissue_type a value.

co to znammneá Somatic variant drivers:

"Somatic variant drivers" se týká genetických variant, které hrají klíčovou roli v procesu vzniku rakoviny a jiných nemocí. Zde je podrobné vysvětlení:

Somatic Variants

  • Somatické varianty jsou genetické změny, které se vyskytují v somatických (ne reprodukčních) buňkách. Tyto změny nejsou přítomné v germinálních buňkách (vajíčka nebo spermie) a tedy se nezdědí.
  • Tyto varianty vznikají v důsledku mutací, které se objevují během života jednotlivce. Mohou být způsobeny různými faktory, jako jsou vystavení chemikáliím, radiaci nebo náhodné chyby při replikaci DNA.

Drivers

  • Driver (spouštěč) je termín, který se používá k označení genetických variant, které aktivně přispívají k rozvoji rakoviny nebo jiného onemocnění. Tyto varianty mohou ovlivnit různé biologické procesy, jako je regulace buněčného cyklu, apoptóza (programovaná smrt buněk), nebo signální dráhy.
  • Driver mutations (mutace spouštěče) jsou tedy ty, které přímo podporují růst a přežití nádorových buněk, na rozdíl od passenger mutations (mutací pasažérů), které jsou přítomné v nádorových buňkách, ale nemají vliv na jejich vývoj.

Přehled

  • Somatic variant drivers jsou tedy somatické genetické změny, které hrají klíčovou roli v nádorovém procesu. Tyto varianty mohou být cílem terapeutických zásahů nebo biomarkerů pro diagnostiku a prognózu onemocnění.

V kontextu výzkumu rakoviny nebo klinické genetiky je důležité identifikovat a pochopit somatické varianty spouštěče, protože mohou ovlivnit léčebné strategie a výsledky pacienta.

Sloupce

  1. VARIANT

    • Popis: Identifikátor genetické varianty.
    • Příklad: BRAF p.V640E, CDKN2A p.A68fs.

  2. VCN (Variant Copy Number)

    • Popis: Počet kopií dané varianty přítomných v genomu. Udává, jak mnoho kopií dané varianty je přítomno.
    • Příklad: 4.1, 2.0.

  3. CN (Copy Number)

    • Popis: Celkový počet kopií určitého genomového segmentu nebo genu. Může odrážet amplifikaci nebo deletaci.
    • Příklad: 6.0, 2.0.

  4. MACN (Minor Allele Copy Number)

    • Popis: Počet kopií menšího alelu (variantního alelu) v porovnání s hlavním alelem.
    • Příklad: 2.0, 0.0.

  5. BIALLELIC

    • Popis: Označuje, zda je varianta biallelická, což znamená, že obě alely (kopie) daného genu obsahují mutaci.
    • Hodnoty: Yes (ano), No (ne).

  6. HOTSPOT

    • Popis: Určuje, zda je varianta známá jako hotspot, což znamená, že se často vyskytuje v daném místě v genomu u různých pacientů.
    • Hodnoty: Yes (ano), No (ne).

  7. DL (Depth of Coverage)

    • Popis: Hloubka pokrytí nebo četnost sekvenování v místě varianty. Udává, kolikrát byla daná oblast genomu sekvenována.
    • Hodnoty: Například 100%, Near, což může znamenat vysokou nebo blízkou kvalitu pokrytí.

  8. CL (Confidence Level)

    • Popis: Úroveň důvěry v detekci varianty. Vyjadřuje, jak jisté jsou výsledky detekce varianty.
    • Hodnoty: Například 100%, což znamená vysokou úroveň důvěry.

  9. PHASE

    • Popis: Určuje fázi varianty nebo její fázi v rámci genetické informace.
    • Hodnoty: Například 5/9 (56%) může indikovat fázi nebo procentuální přítomnost varianty.

  10. ID

    • Popis: Identifikační číslo nebo kód varianty, často používaný pro referenci nebo sledování.
    • Příklad: 1350/1354 (100%).

  11. RNA

    • Popis: Množství RNA, které je k dispozici nebo detekováno pro danou variantu. Indikuje úroveň exprese.
    • Příklad: 0/0, což může znamenat žádnou detekovanou RNA.

  12. DEPTH

    • Popis: Hloubka pokrytí RNA, což ukazuje na množství RNA, které bylo sekvenováno.
    • Příklad: 4725, což znamená počet sekvenovaných čtení.

Chcete-li vytvořit tabulku v balíčku reactable, která bude obsahovat všechny důležité informace ze všech analýz a bude mít možnost rozkliknutí pro zobrazení detailů, můžete použít následující přístup. Tento kód ukazuje, jak můžete vytvořit tabulku s vrchními úrovněmi a podrobnostmi v reactable.

Příklad Kódu

library(shiny)
library(reactable)

# Definice dat
data <- list(
  somatic_variants = data.frame(
    Category = c("Driver variants (4)", "Other potentially relevant variants (2)"),
    Variant = c("BRAF p.V640E", "CDKN2A p.A68fs", "CDKN2A (alt) p.G83fs", "SF3B1 p.P718L",
                "ALB c.1059-52dupT", "MED12 c.3210-6C>T"),
    Frequency = c(0.492, 0.267, 0.25, 0.353, 0.492, 0.267),
    Coverage = c(59, 30, 103, 209, 59, 30),
    Gene_region = c("splice", "splice", "exon", "exon", "splice", "splice"),
    gnomAD_NFE = c(0.09074, 0, 0, 0.01206, 0.09074, 0)
  ),
  germline_variants = data.frame(
    Category = c("Driver variants (0)", "Other potentially relevant variants (4)"),
    Variant = c("CYP2D6 c.506-1G>A", "CYP3A4 c.522-191C>T", "CYP3A4 upstream", "CYP3A5 c.219-237A>G"),
    Frequency = c(0.492, 0.267, 0.25, 0.353),
    Coverage = c(59, 30, 103, 209),
    Gene_region = c("splice", "splice", "exon", "exon"),
    gnomAD_NFE = c(0.09074, 0, 0, 0.01206)
  ),
  fusion_drivers = data.frame(
    Category = c("Fusion drivers (1)", "Other potentially relevant fusions (0)"),
    Fusion = c("KMT2A-MLLT3", NA),
    Arriba_called = c(TRUE, NA),
    StarFus_called = c(TRUE, NA),
    Arriba_confidence = c("high", NA),
    Overall_support = c(35, NA)
  )

# Shiny aplikace
ui <- fluidPage(
  reactableOutput("table")
)

server <- function(input, output) {
  output$table <- renderReactable({
    reactable(
      data = data.frame(
        Group = c(
          "Somatic variant drivers", "Germline variant drivers", "Fusion drivers"
        ),
        Count = c("5 (BRAF, CDKN2A, TERT)", "0", "1 (KMT2A-MLLT3)"),
        stringsAsFactors = FALSE
      ),
      columns = list(
        Group = colDef(name = "Group"),
        Count = colDef(name = "Count")
      ),
      defaultPageSize = 10,
      details = function(index) {
        if (index == 1) {
          reactable(
            data$somatic_variants,
            columns = list(
              Variant = colDef(name = "Variant"),
              Frequency = colDef(name = "Frequency"),
              Coverage = colDef(name = "Coverage"),
              Gene_region = colDef(name = "Gene region"),
              gnomAD_NFE = colDef(name = "gnomAD NFE")
            ),
            defaultPageSize = 10
          )
        } else if (index == 2) {
          reactable(
            data$germline_variants,
            columns = list(
              Variant = colDef(name = "Variant"),
              Frequency = colDef(name = "Frequency"),
              Coverage = colDef(name = "Coverage"),
              Gene_region = colDef(name = "Gene region"),
              gnomAD_NFE = colDef(name = "gnomAD NFE")
            ),
            defaultPageSize = 10
          )
        } else if (index == 3) {
          reactable(
            data$fusion_drivers,
            columns = list(
              Fusion = colDef(name = "Fusion"),
              Arriba_called = colDef(name = "Arriba called"),
              StarFus_called = colDef(name = "StarFus called"),
              Arriba_confidence = colDef(name = "Arriba confidence"),
              Overall_support = colDef(name = "Overall support")
            ),
            defaultPageSize = 10
          )
        }
      }
    )
  })
}

shinyApp(ui, server)

Vysvětlení

  • Data: Uložil jsem tři datové rámce (data frames) do seznamu data pro různé kategorie variant.
  • Shiny aplikace: Definuje uživatelské rozhraní (UI) a server. V serveru renderuji reactable tabulku s hlavními kategoriemi a jejich počty.
  • Details: Funkce details zajišťuje, že když je řádek rozkliknut, zobrazí se detailní tabulka s relevantními informacemi.
